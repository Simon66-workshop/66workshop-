# 66TaskLight M3.8-M4.0 Operations

## Reliability

- Projected UI state expires after 5 seconds. With the 5-second UI refresh loop,
  a stopped projector reaches `projector_stale` within 10 seconds.
- Visible task identity prefers thread, then session, then task/turn identity.
- App-server outages are `probe_unavailable`; they never imply re-Trust.
- Hook-created `done_unverified` tasks auto-release after 30 seconds. Explicit
  managed tasks keep the verification gate.
- Signal IDs are rejected on append and deduplicated during compaction.
- `script/swift_test_verified.sh` runs `swift test` and then executes the shared
  Swift Testing suite. This prevents Command Line Tools from returning a false
  green when `xctest` is absent.

## Observability

- `tasklight_history_index.py` incrementally indexes JSONL streams into SQLite
  WAL mode and publishes `anomaly_summary.json`.
- Event and UI-flow JSONL logs rotate into three bounded archives.
- The diagnostics panel shows history size, duplicate rate, anomaly count, and
  one-hour status transitions.
- Performance budgets are constants in `TaskLightUIPerformanceBudget` and are
  covered by the Swift suite.

## Expansion

- Provider plugins require an explicit enabled manifest and a user-owned,
  non-world-writable executable. They run without a shell or inherited secrets.
- Global shortcuts use Carbon hot keys: Command-Option-T toggles visibility and
  Command-Option-R toggles the expanded panel.
- The WidgetKit extension is a real XcodeGen app-extension target and refreshes
  after sanitized snapshot writes. Team/App Group signing is still required.
- Update manifests use Ed25519 signatures with an external private key.
- Diagnostic ZIPs include only allowlisted health summaries with identifiers,
  paths, messages, and evidence pseudonymized.

Run `script/check_release_readiness.py`. A machine without a signing identity
must remain `blocked_missing_codesign_identity`; local ad-hoc signing is not a
production release.
